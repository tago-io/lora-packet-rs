## Report a security issue

Report security issues privately through [GitHub's security advisory form](https://github.com/tago-io/lora-packet-rs/security/advisories/new). Security issues should not be reported via the public GitHub issue tracker.

## Vulnerability coordination

Confirmed vulnerabilities are coordinated through the [GitHub Security Advisory](https://docs.github.com/en/code-security/security-advisories) opened for the issue. Affected parties include the reporter, direct users of `lora-packet`, and maintainers of dependent crates where relevant.

Downstream maintainers can request inclusion in the coordination of a relevant advisory by commenting on the advisory thread. Participation is at the maintainers' discretion.

## Security advisories

Security issues are disclosed through [GitHub release notes](https://github.com/tago-io/lora-packet-rs/releases) and the [RustSec advisory database](https://github.com/RustSec/advisory-db) (i.e. `cargo audit`).
